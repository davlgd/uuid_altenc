draft RFC 9562 issues

A UUID is 128 bits. The canonical RFC 9562 form spends 36 characters on those bits — fine for logs, wasteful in URLs, invalid as an XML/HTML id. Below: 18 alternative encodings, drawn from the alt uuid encoding methods IETF draft.

UUID accepts hex-dash, hex, Base32hex, NCName-58, Z85

auto only resolves lengths that pin down a single scheme: 36, 32, 26, 25, 24, 23, 20. Lengths 22, 21 and 1–19 are intentionally rejected — variable-width Base36 / Base52 / Base58btc / Base62 alphabets overlap, so the same input could decode to a different UUID under each scheme.
